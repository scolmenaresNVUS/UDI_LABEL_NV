## CRITICAL LABEL FORMAT RULE

**LOT MODE:** Encodes (01) GTIN + (10) Lot + (11) Manufacturing Date. NO expiration date.
**SERIAL MODE:** Encodes (01) GTIN + (17) Expiration + (11) Manufacturing + (21) Serial. WITH expiration date.

**IMPORTANT — Encoding order vs HRI display order are DIFFERENT:**

DataMatrix ENCODING order (optimized for barcode — fixed-length AIs first):
- LOT: `(01)GTIN(11)YYMMDD(10)LOT`
- SERIAL: `(01)GTIN(17)YYMMDD(11)YYMMDD(21)SERIAL`

HRI DISPLAY order (human-readable text printed on the label — identifier right after GTIN):
- LOT: `(01) GTIN` → `(10) LOT` → `(11) YYMMDD`
- SERIAL: `(01) GTIN` → `(21) SERIAL` → `(11) YYMMDD` → `(17) YYMMDD`

The ZPL `^BX` command uses the ENCODING order. The ZPL `^FD` text commands for HRI use the DISPLAY order. These must be implemented separately.

/**
 * All internal measurements are stored in mm.
 * The UI allows the user to toggle between mm and inches.
 * Conversion: 1 inch = 25.4 mm. Display converts on the fly; stored values are always mm.
 */

export type MeasurementUnit = 'mm' | 'in';

export interface BaseElement {
  id: string;
  type: ElementType;
  x_mm: number;            // position always stored in mm
  y_mm: number;            // position always stored in mm
  rotation: 0 | 90 | 180 | 270;
  locked: boolean;
}

export type ElementType = 'datamatrix' | 'hri_text' | 'static_text' | 'line' | 'rectangle';

export interface DataMatrixElementData extends BaseElement {
  type: 'datamatrix';
  moduleSize: number; // cell size in dots, default 4 for 203 DPI
}

export interface HriTextElementData extends BaseElement {
  type: 'hri_text';
  fontSize: number;
  fontFamily: string; // ZPL font: '0','A','B',...,'H'
  lineSpacing: number; // mm between lines
  // Content auto-generated — adapts to lot/serial mode:
  // LOT: 3 lines (01, 10, 11)
  // SERIAL: 4 lines (01, 17, 11, 21)
}

export interface StaticTextElementData extends BaseElement {
  type: 'static_text';
  text: string;
  fontSize: number;
  fontFamily: string;
  bold: boolean;
}

export interface LineElementData extends BaseElement {
  type: 'line';
  endX_mm: number;
  endY_mm: number;
  thickness: number;
}

export interface RectangleElementData extends BaseElement {
  type: 'rectangle';
  width_mm: number;
  height_mm: number;
  borderThickness: number;
  filled: boolean;
}

export type LabelElement =
  | DataMatrixElementData
  | HriTextElementData
  | StaticTextElementData
  | LineElementData
  | RectangleElementData;

/**
 * Label shape can be any rectangle (including square when width === height).
 * Dimensions are ALWAYS stored in mm internally.
 * The UI shows a unit toggle (mm / inches) — conversion is display-only.
 */

  id: string;
  templateGroupId: string;
  version: number;
  isCurrent: boolean;
  name: string;
  description?: string;
  widthMm: number;         // always stored in mm, even if user entered inches
  heightMm: number;        // always stored in mm
  marginMm: number;        // non-printable margin / safe zone, default 1mm
  dpi: number;
  elements: LabelElement[];
  createdBy: string;
  createdAt: string;
}

  { name: '2" × 2" (square)',       widthMm: 50.8,  heightMm: 50.8  },
  { name: '4" × 2" (wide)',         widthMm: 101.6, heightMm: 50.8  },
  { name: '4" × 3" (extra large)',  widthMm: 101.6, heightMm: 76.2  },
  { name: 'Custom',                 widthMm: 0,     heightMm: 0     },

// Unit conversion helpers
export function mmToInches(mm: number): number {
  return Math.round((mm / 25.4) * 1000) / 1000; // 3 decimal places
}

export function inchesToMm(inches: number): number {
  return Math.round(inches * 25.4 * 100) / 100; // 2 decimal places

### Step 4.2 — Designer Canvas (react-konva)

`DesignerCanvas.tsx`:
- Stage sized to label dimensions × display scale (3px per mm)
- White background rectangle = label surface with thin gray border
- **Margin safe zone:** Draw a dashed inner rectangle inset by `marginMm` (default 1mm) from all edges. This represents the printable area. Elements outside this zone risk being clipped by the printer.
- Each element rendered at mm position × scale
- **Draggable elements:** update x_mm/y_mm on drag end
- **Boundary warning:** If any part of an element extends beyond the margin safe zone, highlight that element with a red dashed border and show a small ⚠ icon on it. This is a WARNING, not a block — users may intentionally position elements near edges.
- Click to select (blue border/handles), Konva Transformer for resize
- **Keyboard nudging:**
  - Arrow keys: nudge selected element by 0.25mm

- Delete key removes selected element (with confirmation)
- Ctrl/Cmd+D duplicates selected element
- Optional snap-to-grid toggle:
  - When ON: positions snap to 0.5mm grid (finer than 1mm for precision alignment)

- A toggle in the toolbar: "mm" | "in"

- The toggle affects: properties panel coordinate inputs, canvas ruler markings (if shown), element tooltips

### Step 4.5 — Template Manager & API

**API routes** (`server/src/routes/templates.routes.ts`):

All operations use `templateStore` (JsonStore):
- `GET /api/templates` — read all from templates.json, filter `isCurrent === true`
- `GET /api/templates/:id` — find by id
- `POST /api/templates` — append new template with generated id, templateGroupId, version=1, isCurrent=true
- `PUT /api/templates/:id` — versioned update:
  1. Read all templates
  2. Find current template, get its templateGroupId
  3. Set old version's isCurrent=false
  4. Create new record with same templateGroupId, version+1, isCurrent=true
  5. Write all templates back
  6. Log to audit
- `POST /api/templates/:id/duplicate` — copy with new id and templateGroupId
- `DELETE /api/templates/:id` — remove from array (admin only). Prevent if print jobs reference it.
- `GET /api/templates/:id/versions` — filter all templates by templateGroupId, sorted by version desc

**Template Manager UI:**
- New, Save (creates version), Save As, Load, Duplicate, Delete (admin), Version History

/**
 * Converts mm to dots: dots = mm × (dpi / 25.4), rounded
 */
function mmToDots(mm: number, dpi: number): number {
  return Math.round(mm * (dpi / 25.4));
}

/**
 * Font size (points) to ZPL dots: fontHeightDots = fontSize * dpi / 72
 */
function fontSizeToDots(fontSize: number, dpi: number): number {

 *   ^LL{labelHeightDots}
 *   ^LH0,0
